# 세션 연속성 — MD 3종 운영법

Claude Code 세션이 끝나면 컨텍스트는 사라진다. 다음 세션이 "어제 뭐 했는지"를 알려면 **사람이 읽을 수 있는 MD 3개**에 남긴다. AI가 다음 세션 시작 시 이 3개를 먼저 읽어 컨텍스트를 복원한다.

## 3종 역할 분담

| 파일 | 시간 단위 | 무엇이 들어가나 |
|---|---|---|
| `pairloop/handoff.md` | 세션 단위 (수시간) | 이번 세션에 한 일 / 다음 후보 / 단기 주의사항 |
| `pairloop/known-pitfalls.md` | 프로젝트 생애 (영구) | **2회 이상** 발생한 실수 패턴 |
| `changelog.md` | 마일스톤·의사결정 단위 | 완료한 마일스톤 / 아키텍처 결정 |

추가로 `docs/today.md` (하루 단위 작업 리스트), `pairloop/to-do.md` (백로그) 를 함께 쓴다.

## 세션 시작 규칙

새 세션을 열면 Claude는 아래 순서로 읽는다:

1. `CLAUDE.md` (프로젝트 규약)
2. `pairloop/handoff.md` (최근 2~3세션)
3. `pairloop/known-pitfalls.md` (관련 카테고리만 검색)
4. `pairloop/to-do.md` / `docs/today.md` (오늘 할 일)

이 규칙을 `CLAUDE.md` 에 명시해둬야 Claude가 새 세션에서 자동으로 수행한다. 템플릿 `CLAUDE.md` 는 본 프로토타입에 포함되어 있지 않다 — 프로젝트마다 자기 언어로 쓰는 게 맞다.

## 세션 종료 규칙

Claude에게 "`/pair-hand`" 를 말하면 `pair-hand` 스킬이 발동한다:

1. `pairloop/handoff.md` 읽기
2. 이번 세션 작업 요약 → 최상단에 새 세션 블록 삽입
3. 가장 오래된 세션 블록 제거 (2~3개만 유지)
4. "다음 작업 후보" 갱신
5. 새로 알게 된 **단기 주의사항**만 추가

마일스톤 완료나 아키텍처 결정은 pair-hand가 아니라 `changelog.md` 에 쓴다 (`/pair-hand` 스킬이 3파일 동시 갱신).

## 2회 이상 규칙 — known-pitfalls

단 한 번의 실수는 `pairloop/known-pitfalls.md` 에 올리지 않는다. 2회 이상 같은 패턴이 나오면 그때 등재한다.

**이유**: 1회성 실수는 "그때의 특수 상황"일 가능성이 높다. 2회면 **구조적 함정**이다. 이 구분이 흐려지면 known-pitfalls가 쓰레기통이 된다.

**등재 포맷**: `templates/known-pitfalls.md` 상단 예시를 따른다. 카테고리는 자기 프로젝트에서 실제 2회 이상 난 유형만 만든다.

## 정직한 한계

- **Day 1에는 이 3개가 다 비어 있다.** 실전 효과는 **2~3주 뒤부터** 보인다.
- 축적 콘텐츠는 **외부로 이전되지 않는다** — 다른 프로젝트의 `pairloop/known-pitfalls.md` 가 당신 프로젝트에 유용할 이유가 없다. 포맷만 가져가고 내용은 당신이 쓴다.
- 이 패턴이 효과를 내려면 **매 세션 규율 있게 /pair-hand 를 돌려야 한다.** 3번 연속 잊으면 무너진다.
