# E2E 세션 리포트 — 생성 가이드라인

`/pair-hand` 두 번째 호출 시 자동 생성되는 **통합 E2E 작업 리포트**의 규칙·데이터 출처·디자인 결정을 정의합니다. 실제 동작은 `prototype/skills/pair-hand/SKILL.md`의 통합 리포트 단계가 담당합니다.

리포트가 어떻게 생겼는지 형식만 보고 싶다면 → [`e2e-report-example.md`](./e2e-report-example.md)

---

## 트리거 조건

- **첫 번째 `/pair-hand`** 호출 시: 마커 파일 `pairloop/.handoff-state` 생성 + 짧은 안내만 출력. 리포트는 안 만든다.
- **두 번째 `/pair-hand`** 호출 시: 마커 파일을 감지·삭제하고 리포트 생성 (화면 출력 + md 파일 저장).

판정 로직은 `pair-hand/SKILL.md` 0-b 단계 참조.

---

## 두 가지 출력

### 1. 화면 요약 (즉시 확인)

두 번째 창의 사용자 화면에 박스 형식으로 출력한다. 핵심 통계 + 처리 결과 + 잔여 + 다음 후보를 한눈에 볼 수 있는 분량.

### 2. 저장된 md 파일 (보존·누적)

같은 내용을 다음 위치에 md 파일로 저장한다.

```
pairloop/reports/e2e-report-{YYYY-MM-DD-HHMM}.md
```

- `YYYY-MM-DD-HHMM`: 두 번째 `/pair-hand` 호출 시각 (24시간 표기, 콜론 없음).
  - 예: `e2e-report-2026-04-28-1445.md`
- 폴더가 없으면 자동 생성: `mkdir -p pairloop/reports`
- 파일 본문은 화면 출력과 동일한 박스 + 상단에 메타 헤더(생성 시각·세션 기간·관련 커밋 범위) 추가.
- git 커밋 대상에 포함된다 (pair-hand 7단계의 `git add`에 `pairloop/reports/` 추가).

## 데이터 출처 매핑

리포트 각 줄이 어디서 오는가:

| 리포트 줄 | 출처 |
|-----------|------|
| 시간 범위 시작 | `pairloop/.handoff-state`의 `first-at` |
| 시간 범위 종료 | 두 번째 `/pair-hand` 호출 시각 |
| 사이클 수 | `pairloop/result.md`에서 `🟢/🔴/🟡/🔵 [...] ` 형식의 줄 카운트 |
| 통과/버그/스펙/불안정 분포 | 동일 파일에서 줄 시작 이모지별 카운트 |
| 처리 결과 (✅) | `pairloop/result.md`의 `✅ [...]` 줄 본문에서 키워드 추출 (자동 수정 / 테스트 갱신 / 재시도 통과) |
| 자동 수정 실패 | `❌ 자동 수정 실패` 또는 `❓ 원인 불명` 줄 |
| known-pitfalls 신규 | `pairloop/known-pitfalls.md`에서 시간 범위 내 등재된 `{bugPrefix}-N` ID + 한 줄 요약 |
| git 활동 | `git log --oneline --since="<first-at>"` 결과의 줄 수 + 마지막 줄 |
| 잔여 미해결 | `pairloop/result.md`에서 ✅로 처리되지 않은 🔴/🟡/🔵 줄 |
| 다음 세션 후보 | `pairloop/handoff.md` 최상단 두 블록의 "다음 세션 후보" 합집합 (중복 제거, 우선순위 유지) |

## md 파일 헤더 구성

저장된 리포트 파일의 상단에는 메타 정보를 둔다.

```markdown
# E2E 세션 리포트 — {YYYY-MM-DD HH:MM}

> 자동 생성 by `/pair-hand` (두 번째 창)
> 세션 기간: {시작} ~ {종료} ({경과시간})
> 관련 커밋 범위: {first-commit-hash}..{last-commit-hash}
> 프로젝트: {pairloop.config.json의 project.name}

---

(여기서부터 화면 출력과 동일한 박스 리포트)
```

본문(박스)는 화면과 100% 동일. 헤더만 메타 추가.

## 디자인 결정

- **이모지 + 박스 라인**: 모노스페이스 터미널에서도 알아보기 쉽게 ASCII 박스 + 이모지로 시각적 그룹화.
- **한 섹션이 비면 "(없음)" 명시**: 빈 섹션을 건너뛰지 않고 "(없음)"으로 표시 — 사용자가 "리포트가 잘렸나?" 의심하지 않도록.
- **자동 수정 실패는 잔여 미해결과 함께**: 두 데이터를 한 섹션에 묶어 다음 세션에 무엇이 인계되는지 명확히.
- **다음 세션 후보 = handoff 두 블록 합집합**: 창 A·B 각자의 "다음 세션 후보"를 합쳐 중복 제거. 우선순위는 두 창 합산 등장 횟수 기준.
- **화면과 파일 동일 본문**: 두 출력의 본문은 100% 같다. 사용자가 "화면 본 것과 파일이 다른가?" 의심할 필요 없게.
- **파일은 git 추적**: `pairloop/reports/`는 누적 자료라 commit 대상. 시간이 지나면 retention 분석·세션 패턴 회고에 쓸 수 있다.

## 잘못된 종료 케이스

사용자가 한 창만 `/pair-hand`하고 다른 창은 그냥 닫으면 마커 파일 `pairloop/.handoff-state`가 남는다. 이 경우 다음 세션의 첫 번째 `/pair-hand`가 마커를 발견해 자기를 두 번째로 오인할 수 있다.

대응:
- 사용자가 새 세션 시작 시 마커 파일이 보이면 직접 삭제하거나, `/pair-hand` 시작 시 마커의 `first-at`이 24시간 이상 오래됐으면 리셋한다 (스킬 내부 로직에 추가 가능).
- 매뉴얼 FAQ에 "한 창만 종료하고 닫으면 어떻게 되나요?" 항목으로 안내.
