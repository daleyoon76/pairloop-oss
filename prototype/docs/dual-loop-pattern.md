# 창A/창B 이중 루프 패턴

Claude Code를 두 개의 세션(터미널/창)으로 나누어 **감시 세션(A)**과 **수정 세션(B)**의 역할을 분리한다. 한 세션에서 테스트 실행과 버그 수정을 섞으면 컨텍스트가 오염되어 AI가 "왜 이 코드를 고치는지" 혼동한다. 이 패턴은 그걸 피한다.

## 구성 요소

- **`/pair-watch`** — 창A 전용 감시 루프 스킬. 배포·커밋 변경 감지 + E2E 실행 + 3-way 분류.
- **`/pair-fix`** — 창B 전용 수정 루프 스킬. `pairloop/result.md` 미해결 항목을 위에서부터 처리.
- **`ScheduleWakeup`** — Claude Code 네이티브 도구. 지정한 초 뒤 자동 재기동.
- **공유 신호 파일** — `pairloop/result.md`. 두 세션이 읽고 쓴다.

## 창A — 감시·실행·기록 전담

```
/pair-watch
```

- 할 일: `pairloop/result.md` 미해결 확인 → 없으면 E2E 전체 실행 → 결과 3-way 분류 → 기록 → 4분 대기 → 반복
- 절대 안 할 일: 코드 수정, 실패 원인 직접 분석

**창A 루프 흐름:**
```
시작
  ↓
pairloop/result.md에 미해결 🔴 항목이 있는가?
  → 있으면: 4분 후 재확인 (창B가 처리하는 중)
  → 없으면: E2E 전체 실행 → 결과 분류 → pairloop/result.md 기록
  ↓
(반복)
```

## 창B — 분석·수정 전담

```
/pair-fix
```

- 할 일: `pairloop/result.md`의 🔴·🟡·🔵 항목을 위에서부터 하나씩 처리 → 4분 대기 → 반복
- 🔴 = 코드 수정 + `pairloop/known-pitfalls.md` 2회 이상 조건 체크
- 🟡 = 테스트 파일 업데이트 (프로덕션 코드 수정 금지)
- 🔵 = 3회 재시도 후 2회 이상 실패 시 🔴로 격상
- 절대 안 할 일: 새 E2E 실행 (창A 영역 침범)

## 공유 파일의 동시성

같은 파일을 두 세션이 쓰므로 충돌 가능성이 있다. 완화책:

1. **섹션 분리**: 창A는 `## 미해결` 에 prepend만. 창B는 `## 미해결` 에서 제거 + ✅ 처리.
2. **순차 쓰기**: 두 루프 모두 4분 간격(ScheduleWakeup)이라 동시 쓰기 확률이 낮다.
3. **충돌 시 재시도**: 플레인 파일이므로 락 없음. 충돌 시 다음 사이클에서 재처리.

## 종료 순서

반드시 아래 순서를 따른다. 동시에 실행하면 파일 충돌이 난다.

```
1. 창A: /pair-watch-stop  (현재 사이클 완료 후 루프 종료)
2. 창B: /pair-fix-stop    (남은 항목 모두 처리 후 루프 종료)
3. 창A: /pair-hand       (세션 요약 + 문서 갱신 + git push)
4. 창B: /pair-hand       (창A 완료 후 실행)
```

## 한계

- `/pair-watch`와 `ScheduleWakeup`은 **Claude Code 네이티브**다. Anthropic이 이 기능을 변경하면 패턴이 깨진다.
- 두 세션을 동시에 띄워야 한다 = Claude Code Pro 1개 계정으로 되지만 **토큰 소비 2배**다.
- 1~2인 개발자용 패턴이다. 3인 이상 팀에서는 Git 브랜치 분리가 더 깔끔하다.
- 세션이 닫히면 루프도 종료된다. 24/7 상시 감시는 이 패턴의 범위 밖이다.
