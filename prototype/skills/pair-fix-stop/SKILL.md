창 B의 `/pair-fix` 수정 루프를 안전하게 종료한다.
**남아있는 미처리 항목은 모두 처리한 뒤에 종료한다.**

---

## 실행 절차

### 1. 미처리 항목 전체 목록 확인

`pairloop/result.md`를 읽어 ✅로 처리되지 않은 모든 🔴 / 🟡 / 🔵 항목을 위에서부터 순서대로 나열한다.

```
잔여 항목: 🔴{N} 🟡{N} 🔵{N}
```

### 2. 위에서부터 순서대로 모두 처리

`/pair-fix`의 1-b / 1-c / 1-d 로직을 그대로 적용한다.

- **🔴**: 원인 파악 → 코드 수정 → 재실행 통과 확인 → ✅ 표시 + 커밋 해시 기록
- **🟡**: 스펙 변경 확인 → 테스트 파일 업데이트 → 재실행 통과 확인 → ✅ 표시
- **🔵**: 3회 재시도 → 2회 이상 실패 시 🔴로 격상 후 처리, 그렇지 않으면 🟢로 정리

처리 1건이 끝날 때마다 진행 상황을 한 줄로 출력:
```
처리 완료 ({현재}/{전체}): {항목 한 줄 요약}
```

### 3. 처리 불가 항목

원인을 특정할 수 없거나 수정에 실패한 항목은 강제로 ✅ 표시하지 않는다.
대신 항목 끝에 `❓ 원인 불명` 또는 `❌ 자동 수정 실패: <사유>`를 덧붙여 다음 세션에 인계한다.

### 4. 루프 종료 및 메시지 출력

모든 항목 처리(또는 인계 표시)가 끝나면 TaskStop으로 종료하고 출력:

```
수정 루프가 종료되었습니다.
다음 단계: 창 A에서 /pair-hand 를 먼저 실행해 주세요.
```

---

## 구현 참고

- 처리 도중 새 항목이 `pairloop/result.md`에 추가되어 있다면 (창 A가 아직 살아있을 수 있음) 그것까지 모두 처리한 뒤 종료한다.
- 무한 루프 방지: 동일 항목을 3회 시도해도 통과시키지 못하면 `❌ 자동 수정 실패`로 표시하고 다음으로 넘어간다.
