이 스킬은 **창 B 전용**이다.
창 A(`/pair-watch`)가 `pairloop/result.md`에 기록한 실패 항목을 위에서부터 자동으로 처리한다.

---

## 0. 설정 로드

프로젝트 루트의 `pairloop/pairloop.config.json`을 읽는다.
필요한 값:
- `test.command` — 재시도 시 사용할 테스트 명령

**파일이 없으면 즉시 중단하고 출력:**
```
pairloop/pairloop.config.json을 찾을 수 없습니다.
프로젝트 루트에서 install-pairloop 스크립트를 먼저 실행해 주세요.
```

---

## 1. 메인 루프

아래 순서를 무한 반복한다. 4분 대기는 ScheduleWakeup(delaySeconds=240)을 사용한다.

### 1-a. 중단 마커 확인 + 항목 선택

**루프 진입 시 가장 먼저** `pairloop/.pairloop-stop` 파일이 있는지 확인한다.
- **있으면** → 파일을 삭제하고 즉시 루프를 종료한다:
  ```
  ✅ handoff 완료 감지 — 수정 루프를 종료합니다.
  ```
- **없으면** → 계속 진행한다.

`pairloop/result.md`를 읽어 위에서부터 첫 번째 🔴 / 🟡 / 🔵 (✅ 처리되지 않은 항목)을 찾는다.

**처리 대상이 없으면:**
아래를 출력하고 ScheduleWakeup(delaySeconds=240)으로 4분 대기 후 1-a로 복귀.
```
모든 항목 처리됨. 4분 후 재확인.
```

**처리 대상이 있으면:**
가장 위 항목을 선택하고 종류에 따라 1-b / 1-c / 1-d로 분기.
```
🔧 처리 시작: {항목 줄 그대로}
```

### 1-b. 🔴 항목 처리 (실제 버그 수정)

1. `pairloop/known-pitfalls.md`에서 동일 패턴 검색 → 이전 수정 기록이 있으면 참고.
2. 항목 바로 아래 `콘솔:` 줄이 있으면 명시된 `pairloop/console-logs/*.txt` 파일을 읽어 브라우저 에러 상세를 확인한다. 관련 코드 파일과 최근 `git diff`를 함께 확인해 원인을 파악한다.
3. 원인을 한 줄로 출력한다.
4. **코드를 수정**한다 (테스트 파일이 아니라 실제 프로덕션 코드).
5. 수정 후 `pairloop/pairloop.config.json`의 `test.command`로 해당 시나리오만 재실행해서 통과 확인.
6. `pairloop/result.md`의 해당 줄을 ✅로 변경하고 수정 커밋 해시를 추가:
   ```
   ✅ [YYYY-MM-DD HH:MM] <시나리오 이름> — <수정 요약> (커밋 <해시7자리>)
   ```
7. `pairloop/known-pitfalls.md`의 해당 항목에 수정 방법(`수정:` 줄)을 업데이트.

수정이 확실하지 않거나 원인을 특정할 수 없으면 코드를 건드리지 말고 항목 끝에 `❓ 원인 불명` 을 덧붙인 뒤 다음 항목으로 넘어간다.

### 1-c. 🟡 항목 처리 (스펙 변경에 따른 테스트 업데이트)

1. 커밋 메시지·`pairloop/to-do.md`에서 어떤 스펙 변경이 일어났는지 확인.
2. 해당 시나리오의 **테스트 파일**을 새 스펙에 맞게 업데이트한다 (프로덕션 코드는 건드리지 않는다).
3. 재실행으로 통과 확인.
4. `pairloop/result.md`의 해당 줄을 ✅로 변경:
   ```
   ✅ [YYYY-MM-DD HH:MM] <시나리오 이름> — 테스트를 새 스펙에 맞게 업데이트 (커밋 <해시7자리>)
   ```

### 1-d. 🔵 항목 처리 (간헐 실패 재검증)

1. 동일 조건으로 해당 시나리오를 **3회 재실행**한다.
2. 결과에 따라 분기:
   - **2회 이상 실패** → 🔴로 격상. pairloop/result.md의 해당 줄을 다음과 같이 변경하고 1-b 로직으로 처리:
     ```
     🔴 [YYYY-MM-DD HH:MM] <시나리오 이름> — 3회 중 N회 실패, 실제 버그로 격상
     ```
   - **1회 이하 실패 (즉, 2회 이상 통과)** → 🟢로 정리:
     ```
     🟢 [YYYY-MM-DD HH:MM] <시나리오 이름> — 재시도 통과 (간헐성 확인)
     ```

### 1-e. 루프 복귀

처리 1건 완료 후 1-a로 즉시 돌아간다 (대기 없음).
처리 대상이 없을 때만 4분 대기.

---

## 2. 중지

- `/pair-fix-stop` 실행 시 → 남은 🔴·🟡·🔵 항목을 모두 처리한 뒤 종료 (해당 스킬이 처리)
- 사용자가 "중지", "스탑", "stop" 입력 시 → 처리 중인 1건이 끝나면 즉시 TaskStop으로 종료

---

## 구현 참고

- 한 번에 하나씩만 처리한다. 여러 항목을 동시에 건드리지 않는다.
- 창 A가 새 실패 항목을 `pairloop/result.md` **상단**에 추가하고, 이 스킬은 위에서부터 처리하므로 동시 실행해도 충돌하지 않는다.
- 🔴 처리 시 절대 테스트 파일을 고치지 않는다 (그건 🟡 처리에서만 한다). 테스트를 통과시키기 위한 테스트 수정은 금지다.
