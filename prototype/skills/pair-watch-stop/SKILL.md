창 A의 `/pair-watch` 감시 루프를 안전하게 종료한다.
**진행 중인 E2E 사이클은 절대 중간에 끊지 않는다.**

---

## 실행 절차

### 1. 현재 상태 확인

`/pair-watch` 루프가 어떤 상태인지 판단한다.

- **E2E 실행 중**: 해당 사이클이 완전히 끝나고 분류·기록이 마무리될 때까지 대기한다.
- **4분 대기 중 (미해결 항목 재확인 대기)**: 즉시 종료 단계로 넘어간다.
- **시나리오 파일 대기 중 (30초 폴링)**: 즉시 종료 단계로 넘어간다.

### 2. 사이클 완료 대기

E2E가 실행 중이라면 다음을 보장한 뒤에만 다음 단계로 진행한다.
- 모든 시나리오 실행 완료
- 3-way 분류 완료
- `pairloop/result.md` 기록 완료

### 3. 루프 종료

`/pair-watch`의 메인 루프를 TaskStop으로 종료한다.
ScheduleWakeup이 예약되어 있으면 더 이상 새 사이클이 진입하지 못하도록 종료 플래그를 남긴다.

### 4. 종료 메시지 출력

```
감시 루프가 종료되었습니다.
다음 단계: 창 B에서 /pair-fix-stop 을 실행해 주세요.
```

---

## 구현 참고

- 절대 강제 종료(kill)하지 않는다. 진행 중인 테스트 결과가 분류·기록되지 못하면 다음 세션이 잘못된 상태로 시작된다.
- 4분 대기 중인 ScheduleWakeup은 깨우지 않고 종료해도 무방하다 (E2E가 실행 중이 아니므로).
